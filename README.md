学生信息管理
​​(1) 添加学生​​：允许用户输入新学生的个人信息，包括学号、姓名、年龄、家庭地址、电话和邮箱。确保所有必填字段都得到正确填写，并进行基本的数据验证（例如，确保学号唯一性、电话号码格式正确）。
​​(2) 编辑学生信息​​：用户可以根据学号查找并修改现有学生的信息。系统应提供确认机制，以防止误操作导致数据丢失或错误修改。
​​(3) 删除学生记录​​：提供安全的方式让用户根据学号删除不再需要的学生记录。同样地，应设置确认步骤以避免意外删除。
​​(4) 查询学生信息​​：通过学号快速查找特定学生的信息。也可通过多种条件（如学号、姓名、电话等）查询。
​​(5) 浏览所有学生​​：提供一个功能，让用户能够查看系统中所有学生的概览信息，便于全面了解当前学生的情况。
​​(6) 排序与筛选​​：用户可以根据不同的属性（如按学号升序/降序、按姓名首字母等）对结果进行排序，也可以应用筛选器来缩小查询范围，例如只显示某一年龄段的学生。
2.2.2 数据持久化
​​(1) 保存至文件​​：为了保证数据的安全性和持久性，所有学生信息应在每次修改后自动保存到磁盘上的文件中（例如文本文件.txt）。此外，还应该提供手动保存选项，以便在必要时立即保存更改。
​​(2) 加载数据​​：当程序启动时，自动从磁盘读取最新的学生信息数据，恢复到上次关闭前的状态。
2.3 性能需求
​​(1) 响应时间​​：确保系统的操作（特别是查询操作）具有较快的响应速度，一般情况下应在1秒内完成。
​​(2) 数据一致性​​：维护数据的一致性和完整性，确保不会因为程序崩溃或其他原因而导致数据损坏或丢失。
2.4 用户需求
​​(1) 易用性​​：设计简洁明了的命令行界面或图形用户界面（GUI），使用户无需复杂的培训即可轻松上手使用。
​​(2) 可靠性​​：系统需稳定运行，具备良好的异常处理能力，即使遇到未预见的问题也能尽量保护数据不被破坏。
